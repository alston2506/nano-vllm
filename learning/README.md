# nano-vllm 持久学习档案

这个目录用于存放你学习 nano-vllm 和大模型推理过程中的文档、练习代码和进度记录。以后你在这个项目下打开 Codex，只要说“开始学习”或“继续学习”，Codex 会先读取 `learning/progress.md`，然后从上次停下的位置继续辅导。

## 目录结构

- `progress.md`：唯一的学习进度入口。每次学习结束都要更新。
- `curriculum.md`：长期学习路线图，从推理基础到 nano-vllm 源码、优化和实战改造。
- `docs/`：每一课的讲义、概念解释和源码阅读笔记。
- `exercises/`：可运行的小练习，优先用轻量代码模拟核心机制。
- `notes/`：你的自由笔记、问题清单、错题和复盘。

## 每次学习会话的流程

1. 读 `progress.md`，确认当前阶段、当前课和下一步。
2. 用 3-5 分钟回顾上一课最重要的概念。
3. 先讲背景知识：这个问题为什么存在，它在大模型推理中解决什么痛点。
4. 再讲核心直觉：用小例子、类比或伪代码解释，不急着打开源码。
5. 然后进入 nano-vllm 源码：只看和当前概念直接相关的文件和函数。
6. 最后回扣知识：用源码解释刚才的背景概念，让你用自己的话复述。
7. 安排实际代码编写操练：优先写一个小模拟、小实验或小改造，再运行观察结果。
8. 根据你的提问和卡住的地方，更新 `learning/docs/` 中对应课程讲义。
9. 更新 `progress.md`，明确下次从哪里继续。

## 教学顺序原则

默认顺序必须是：

```text
背景问题 -> 核心直觉 -> 小例子 -> nano-vllm 源码 -> 回扣知识 -> 代码操练 -> 练习/复述
```

不要一上来就读源码。源码是用来验证和固定概念的，不是学习的第一入口。

## 代码操练规则

每一课尽量安排一个实际代码编写环节。代码操练优先放在 `learning/exercises/`，形式可以是：

- 从零写一个小模拟，帮助理解概念。
- 修改已有练习，观察输出变化。
- 给 nano-vllm 附近写一个不影响主项目的调试/分析脚本。
- 在理解足够后，做一个很小的源码改造或测试。

每个练习尽量包含：

- 练习目标：这段代码要帮助理解什么。
- 要编辑或运行的文件。
- 预期观察：运行后应该看见什么。
- 思考题：如何把练习现象对应回 nano-vllm。

## 讲义维护规则

`learning/docs/` 不是一次性写完的静态文档，而是随学习过程增长的个人教材。

每次学习中，只要出现下面情况，Codex 都需要更新对应讲义：

- 讲了新的背景知识。
- 你提出了一个能帮助复习的问题。
- 某个概念解释得不够细，需要补例子。
- 源码阅读中发现了新的关键路径。
- 练习暴露出容易混淆的点。

更新讲义时优先补充：

- 背景和动机：为什么需要这个概念。
- 最小定义：它到底是什么。
- 直觉例子：先让概念能被抓住。
- nano-vllm 对应文件和函数。
- 常见误区和检查点。

## 结束学习会话

当你说“不学了”“今天到这”“先不学了”“停止学习”等表达时，Codex 需要：

1. 更新 `learning/progress.md`，记录本次学到哪里。
2. 检查当前 git 分支和工作区状态。
3. 在 `main` 分支上提交本次学习相关改动。
4. 推送到远端 `origin/main`。

如果推送需要网络或凭据授权，Codex 会按环境要求请求批准。

## 学习目标

你要从零基础入手，通过 nano-vllm 这份小而完整的代码，逐步掌握 vLLM，最后形成大模型推理的基本工程能力。

这里的“零基础”指的是：不默认你已经懂机器学习、Transformer、GPU、CUDA、PyTorch 深度学习栈、LLM 推理系统或 vLLM。只默认你有基础 Python 和 C++ 能力。

## 学习方式：源码主线 + 前置知识支线

学习不会只局限在本项目源码。每当 nano-vllm 中出现你还没系统学过的概念，会先补一段必要前置知识，再回到源码中验证它。

前置知识包括但不限于：

- 机器学习/深度学习最小基础：模型、参数、张量、矩阵乘法、训练和推理的区别；反向传播只做理解推理所需的部分。
- Transformer 基础：embedding、self-attention、MLP、layer norm、residual、causal mask。
- LLM 推理基础：tokenizer、logits、采样、自回归生成、prefill/decode。
- GPU 与 PyTorch 执行模型：CUDA 张量、显存、kernel、stream 的入门直觉。
- 系统知识：批处理、调度、缓存、分页思想、吞吐和延迟。
- vLLM 背景：PagedAttention、continuous batching、KV cache 管理、prefix cache。

原则是：前置知识只学到能读懂和修改 nano-vllm 的程度，然后立刻回到代码。

## 当前主线

我们会围绕一条真实推理请求的生命周期学习：

`prompt -> tokenizer -> Sequence -> Scheduler -> prefill -> KV cache -> decode -> sampler -> output`

这条线会反复出现。每学一个模块，都要回答三个问题：

1. 它解决了什么推理问题？
2. 它在 nano-vllm 中对应哪些文件和函数？
3. 如果没有它，系统会慢在哪里、错在哪里，或者无法扩展到哪里？
